
# 🛠️ Tutoriel Complet d’Utilisation de l’API HelpConfort

Ce guide a pour objectif de vous permettre de comprendre la structure de l'API HelpConfort, les liens entre les entités, et de pouvoir extraire des données métiers clés : techniciens, RDVs, devis, factures, clients, CA, etc.

---

## 📦 1. STRUCTURE DES DONNÉES (TABLES/API)

| Table / API Endpoint                   | Rôle                                                                 |
|----------------------------------------|----------------------------------------------------------------------|
| `/api/apiGetUsers`                    | Liste des utilisateurs (techniciens & membres du bureau)            |
| `/api/apiGetClients`                  | Liste des clients (type `client` ou `commanditaire`)                |
| `/api/apiGetDevis`                    | Devis associés aux projets/interventions                            |
| `/api/apiGetFactures`                 | Factures liées aux devis/interventions                              |
| `/api/apiGetProjects`                 | Contient toute la donnée d’un projet (RDV, anomalies, actions, etc.)|

---

## 🔗 2. CLÉS PRIMAIRES ET LIENS ENTRE LES TABLES

| Clé                  | Description                                               | Utilisé dans                          |
|----------------------|-----------------------------------------------------------|----------------------------------------|
| `projectId`          | ID d’un projet                                             | Projets, Devis, Factures              |
| `userId`             | ID d’un utilisateur (tech, bureau)                        | Users, History, RDVs, Factures        |
| `clientId`           | ID du client commanditaire                                | Clients, Projets, Devis, Factures     |
| `refId` / `refDevisId` | Référence entre projet / devis / facture                  | Devis → Factures, Factures → Projets  |
| `uid`                | Identifiant unique d’une ligne de devis/facture           | Factures, Devis                        |

---

## 🧩 3. EXEMPLES DE REQUÊTES ET CORRÉLATIONS

### 📌 Quel technicien est intervenu au premier RDV ?
- Endpoint : `apiGetProjects`
- Champ : `visites[0].usersIds`

### 🔧 Est-ce que ça a été dépanné au 1er RDV ?
- `biDepan.items[0].isWorkDone == true`

### 📄 Y a-t-il eu un devis ?
- Vérifier dans `/api/apiGetDevis` un enregistrement avec `projectId` ou `refInterventionId` égal à celui du projet.

### 🛠️ Technicien(s) ayant fait les travaux :
- `biTvx.items[x].techDiag`, `techTimeStart`, `techTimeEnd`, `isWorkDone`, `usersIds`

### 📊 Chiffre d'affaires par technicien :
- À partir de `/api/apiGetFactures`, répartir `totalHT` ou `totalTTC` selon `nbTechs` et les `userIds` du chantier.

## 🧠 6. COMPÉTENCES TECHNICIENS & UNIVERS

- Les techniciens (`/api/apiGetUsers`) ont un champ `universes` : liste des univers associés.
- Les projets (`/api/apiGetProjects`) ont aussi des `universes`.

Cela permet de **matcher expertises et types d’intervention**.

---

## 📅 7. RECONSTITUTION DE PLANNING

- Extraire tous les `biDepan.items` et `biTvx.items` avec :
  - `techTimeStart`, `techTimeEnd`, `usersIds`
- Cross-check avec `/api/apiGetUsers` pour affectation
- Idéal pour créer une **planification intelligente** ou Gantt

---

## 📈 8. INDICATEURS CLÉS À EXTRAIRE

- **CA par jour/semaine/mois/année** → cumuler `factures.totalHT` par `dateReelle`
- **CA par univers** → croiser projet `universes` + facture liée
- **CA par apporteur** → `commanditaireId` ou `data.commanditaireId`
- **Productivité technicien** = CA / nb d'heures travaillées

---

## 🔍 9. UTILISER LES CHAMPS `history` POUR LE SUIVI

Chaque entité a un historique de modifications :
```json
"history": [
  {
    "userId": 8,
    "labelKind": "Validation d'une tâche : URGENT",
    "dateModif": "2025-03-19T19:33:26"
  }
]
```

---

## ✅ 10. RECOMMANDATIONS TECHNIQUES

- Utiliser un outil comme Metabase, Superset, Power BI ou Grafana
- Stocker les exports pour analyse big data
- Utiliser des IDs uniques (`uid`, `userId`, `projectId`) pour les jointures SQL
- Si besoin de cache : Redis ou Mongo pour stockage rapide intermédiaire

---

## 🧩 Exemple de Corrélation
Pour connaître les techniciens d’un projet :
1. Aller dans `/api/apiGetProjects`
2. Extraire les `usersIds` de `visites`, `biDepan.items`, `biTvx.items`
3. Cross-check avec `/api/apiGetUsers`

Pour déterminer le CA du projet :
1. Aller dans `/api/apiGetFactures` filtré par `projectId`
2. Diviser `totalHT` par le nombre de `nbTechs` associés
