## 1. Vision d'Ensemble

### 1.1 Qu'est-ce que GLOBAL / Apogée ?

GLOBAL / Apogée est une **plateforme SaaS intégrée** conçue pour les agences du réseau HelpConfort. Elle centralise :

- 📚 **La documentation métier** (guides Apogée, apporteurs, procédures)
- 🤖 **Un assistant IA intelligent** (Mme MICHU)
- 📊 **Le pilotage d'activité** (KPIs, indicateurs, tableaux de bord)
- 🎫 **Le support utilisateur** (chat, tickets, FAQ)
- 🏢 **La gestion réseau** (multi-agences, redevances, comparatifs)

### 1.2 À qui s'adresse la plateforme ?

| Profil | Usage principal |
|--------|-----------------|
| **Assistante** | Documentation, support, formation |
| **Technicien** | Guides techniques, planning |
| **Commercial** | Guide apporteurs, procédures |
| **Dirigeant** | KPIs agence, pilotage, équipe |
| **Tête de réseau** | Multi-agences, statistiques, redevances |
| **Administrateur** | Configuration, utilisateurs, système |

### 1.3 Problématiques résolues

✅ Centralisation de la documentation éparpillée  
✅ Formation continue et autonome des équipes  
✅ Accès rapide aux informations métier  
✅ Pilotage en temps réel de l'activité  
✅ Support réactif et traçable  
✅ Vision consolidée pour la direction réseau

## 2. Modules Fonctionnels

### 2.1 Help! Academy — Centre de Documentation

#### Public cible
Tous les collaborateurs du réseau.

#### Objectifs métier
- Maîtriser le logiciel CRM Apogée
- Comprendre les processus métier
- Former les nouveaux arrivants
- Accéder aux procédures à jour

#### Fonctionnalités principales

| Fonctionnalité | Description |
|----------------|-------------|
| **Guide Apogée** | Documentation complète du CRM (clients, dossiers, devis, factures, interventions) |
| **Guide Apporteurs** | Ressources pour la gestion des apporteurs d'affaires |
| **Base Documentaire** | Documents internes, procédures, modèles |
| **Favoris** | Sauvegarde des sections fréquemment consultées |
| **Recherche** | Recherche textuelle dans tous les contenus |

#### Parcours utilisateur type

1. Connexion à la plateforme
2. Accès à Help! Academy depuis le menu
3. Sélection du guide approprié (Apogée, Apporteurs, Documents)
4. Navigation par catégories ou recherche
5. Consultation de la section souhaitée
6. Ajout aux favoris si consultation fréquente

### 2.2 Assistant IA — Mme MICHU

#### Public cible
Tous les utilisateurs ayant des questions métier.

#### Objectifs métier
- Répondre instantanément aux questions
- Guider vers les procédures appropriées
- Réduire la charge du support humain
- Disponibilité 24/7

#### Fonctionnalités principales

| Fonctionnalité | Description |
|----------------|-------------|
| **Chat IA contextuel** | Réponses basées sur la documentation |
| **Multi-contextes** | Apogée, Apporteurs, HelpConfort, Métier |
| **Citations sources** | Liens vers les sections de documentation |
| **Escalade humain** | Transfert vers un conseiller si nécessaire |
| **Création ticket** | Transformation en demande formelle |

#### Parcours utilisateur type

1. Clic sur l'icône chat (coin inférieur droit)
2. Formulation de la question en langage naturel
3. Réception de la réponse avec sources
4. Si besoin : demande de précision ou escalade

#### Bonnes pratiques

- ✅ Poser des questions claires et contextualisées
- ✅ Préciser le module Apogée concerné
- ✅ Consulter les liens sources proposés
- ❌ Ne pas poser plusieurs questions à la fois

### 2.3 Pilotage Agence — Indicateurs & KPIs

#### Public cible
Dirigeants d'agence, assistantes de direction.

#### Objectifs métier
- Suivre l'activité en temps réel
- Identifier les tendances
- Piloter les objectifs commerciaux
- Analyser les performances

#### Fonctionnalités principales

| Fonctionnalité | Description |
|----------------|-------------|
| **Dashboard général** | Vue synthétique des KPIs clés |
| **Indicateurs Apporteurs** | CA, dossiers, taux de conversion par apporteur |
| **Indicateurs Univers** | Performance par univers (plomberie, électricité, etc.) |
| **Indicateurs Techniciens** | CA, interventions, productivité |
| **Indicateurs SAV** | Taux SAV, délais, satisfaction |
| **Filtres temporels** | Jour, semaine, mois, année, 12 mois glissants |

#### Parcours utilisateur type

1. Accès à Pilotage Agence
2. Sélection de la période souhaitée
3. Consultation du dashboard général
4. Drill-down sur un indicateur spécifique
5. Export ou partage si nécessaire

#### Données affichées

- **CA** : Chiffre d'affaires facturé
- **Dossiers** : Nombre de projets créés/clôturés
- **Transformation** : Taux devis → facture
- **SAV** : Taux et délai de traitement
- **Productivité** : CA/technicien, interventions/jour

## 3. Règles Transverses

### 3.1 Gestion des Rôles

La plateforme utilise 7 niveaux de rôles hiérarchiques :

| Niveau | Rôle | Droits principaux |
|--------|------|-------------------|
| N0 | Visiteur | Lecture guides |
| N1 | Utilisateur | + Support, favoris |
| N2 | Dirigeant agence | + Pilotage agence |
| N3 | Animateur réseau | + Multi-agences |
| N4 | Directeur/DG | + Redevances, config |
| N5 | Admin plateforme | + Administration |
| N6 | Superadmin | Accès total |

### 3.2 Règles de priorisation (Tickets)

| Priorité | Délai SLA | Cas d'usage |
|----------|-----------|-------------|
| **Bloquant** | 4h | Système inaccessible |
| **Urgent** | 8h | Bug critique |
| **Important** | 24h | Dysfonctionnement |
| **Normal** | 48h | Question, demande |
| **Amélioration** | 72h | Suggestion |

### 3.3 Bonnes pratiques

1. **Documentation** : Consulter les guides avant de contacter le support
2. **Chat IA** : Premier réflexe pour les questions
3. **Précision** : Être clair et détaillé dans les demandes
4. **Captures** : Joindre des captures d'écran si pertinent
5. **Suivi** : Répondre aux demandes de précision
