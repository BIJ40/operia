# authorization/v2-role-ceiling-management-rules
```
Role management in V2 follows strict ceiling rules enforced both client-side and server-side. Minimum N3 (franchisor_user) required to manage users - N0-N2 cannot create/edit users. Maximum assignable role equals the manager's own level: N3 can assign N0-N3, N4 can assign N0-N4, N5 can assign N0-N5, N6 can assign all roles. Legacy admins (user_roles.role='admin') bypass these restrictions. Key functions: canManageUsers(role) checks N3+ eligibility, getAssignableRoles(role) returns filtered role list. Server-side validation in create-user edge function rejects privilege escalation attempts with explicit error logging. UI displays warning "Vous pouvez assigner des rôles jusqu'à N{X}" when role options are limited.
```

# ui/guide-tiles-unified-design-system
```
Guide pages (ApogeeGuide, HelpConfort, ApporteurGuide) use a unified tile design system matching the menu's selected item style. Active tiles: bg-gradient-to-r from-helpconfort-blue/10 via-helpconfort-blue/5 to-transparent, border-helpconfort-blue/20, border-l-4 border-l-helpconfort-blue (left accent bar), hover:from-helpconfort-blue/15 hover:via-helpconfort-blue/8 hover:border-helpconfort-blue/30 hover:shadow-lg. Empty/disabled tiles: bg-muted/50, border-muted-foreground/30, border-l-muted-foreground/50, opacity-60. "En cours" badges use bg-helpconfort-blue (not orange). "Nouveau" badges remain green. This design eliminates orange colors entirely and creates visual consistency between sidebar navigation and guide content tiles.
```
